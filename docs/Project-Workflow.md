# Project Workflow

# Mobile Banking Application

## Software Development Life Cycle (SDLC)

The Mobile Banking Application is developed following a structured Software Development Life Cycle (SDLC) integrated with Agile and DevOps practices.

---

## Phase 1 – Requirement Analysis

* Gather functional requirements.
* Gather non-functional requirements.
* Identify project scope.
* Prepare Software Requirement Specification (SRS).

Deliverable:

* Requirements.md

---

## Phase 2 – System Design

* Design Three-Tier Architecture.
* Design database structure.
* Prepare UML and architecture diagrams.

Deliverables:

* Architecture.md
* Diagrams

---

## Phase 3 – Development

Frontend

* Flutter Development

Backend

* Spring Boot REST APIs

Database

* MySQL Schema

Deliverables:

* Source Code

---

## Phase 4 – Testing

* Unit Testing
* API Testing
* Functional Testing
* Integration Testing

Tools:

* JUnit
* Postman

---

## Phase 5 – DevOps

* Source Code Management using Git
* Remote Repository using GitHub
* Docker Containerization
* Jenkins CI/CD Pipeline

Deliverables:

* Docker Image
* Jenkins Pipeline

---

## Phase 6 – Deployment

* Deploy application on AWS EC2.
* Configure production environment.

---

## Phase 7 – Monitoring

* Collect metrics using Prometheus.
* Visualize dashboards using Grafana.
* Monitor application performance and health.
